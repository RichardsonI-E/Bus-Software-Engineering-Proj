# Bus-Software-Engineering-Proj
This is the coding project for the Software Engineering Project created by Group 6.
Diclaimer that this project is entirely intended for Education purpsoes only and is not meant for real world application. 

Important information to understand how the code works overall. The program can be started from any .java file within src/main/java folders. If there seems to be an issues starting the code, run MainApp.java within the java folder.

Overall organization:
   A majority of the code is held within the java file and each aspect is organized into is own folder. All files with the .json is the storage folder for any data that is inputed into the program. 
  The Componets folder is the creation of the hambuger symbol and the methods it has while also including a listner for the mouse to track what the user is clicking. 
  
  The json folder is where the storage of the buses, stations, and users data is held that is created within the code.
  The permissions folder just simply creates the specific authority for overall admin (access anything), Bus Manager (can access the Buses data), and Station Manager (can accesss station data).
  
  The primary folder holds the Bus, Station, and User.java files which holds a the core concepts and methods of the program. 
  The screens folder holds the design of every screen you will see within the program and how they function when accessed.
  The MainApp.java file is the file that imports everything and creates the disclaimer page. Run this specfic file if the running the other files does not work.

  The Resources folder just holds the Hambuger png file.
